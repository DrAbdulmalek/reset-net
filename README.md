## المشكلة

عند انتهاء صلاحية Outline VPN (أو أي VPN يعمل عبر TUN)، تبقى بقايا إعداداته في النظام: واجهات TUN معلّقة، قواعد iptables/nftables تحوّل حركة المرور، **قواعد policy routing (`ip rule`) تحوّل كل الترافيك إلى جدول توجيه ميت عبر النفق المُعطَّل**، مسارات DNS مشوّهة، وربط `/etc/resolv.conf` مكسور. النتيجة: **الجهاز لا يتصل بالإنترنت حتى بعد إيقاف الـ VPN**.

## الحل

سكربت `reset-net` ينظّف كل طبقات الشبكة المتأثرة من جذورها في 9 خطوات منظّمة، ثم يعيد تفعيل الاتصال ويتحقق من عمله.

## التثبيت

### الطريقة الأسهل — AUR (أي مستخدم مانجارو/أرش)

```bash
yay -S reset-net
```

> يتطلّب حزمة `yad` للأيقونة في علبة النظام: `sudo pacman -S yad`

### التثبيت اليدوي

```bash
git clone https://github.com/DrAbdulmalek/reset-net.git
cd reset-net
sudo bash install.sh
```

### ماذا يفعل المُثبّت؟

| الخطوة | الفعل |
|--------|-------|
| 1 | نسخ `reset-net` إلى `/usr/local/bin/` |
| 2 | نسخ `reset-net-gui` (الواجهة الرسومية) إلى `/usr/local/bin/` |
| 3 | تثبيت ملف `.desktop` في قائمة التطبيقات |
| 4 | تثبيت سياسة Polkit للمصادقة الرسومية |
| 5 | إنشاء قاعدة `sudoers` لتشغيل `reset-net` بدون كلمة مرور من الطرفية |
| 6 | تثبيت أيقونة علبة النظام + تشغيل تلقائي عند تسجيل الدخول |
| 7 | تحديث قاعدة بيانات التطبيقات |

## الاستخدام

### 1. أيقونة علبة النظام — الأسهل

بعد التثبيت، تظهر أيقونة 🌐 في علبة النظام (بجانب الساعة) تلقائياً عند تسجيل الدخول:

- **نقرة مزدوجة (يسار)** أو **نقرة وسطى**: تشغيل `reset-net` فوراً (يطلب كلمة مرور واحدة عبر Polkit)
- **نقرة يمين**: قائمة منسدلة:
  - إعادة ضبط الشبكة الآن
  - عرض آخر سجل
  - خروج

> لبدء الأيقونة فوراً بدون إعادة تسجيل الدخول: `reset-net-tray &`

### 2. من الطرفية (Terminal)

```bash
reset-net
```
يعمل مباشرة بدون كلمة مرور بفضل إعداد NOPASSWD في sudoers.

### 3. من قائمة التطبيقات (GUI)

ابحث في قائمة التطبيقات عن:
- **Reset Network** (إنجليزي)
- **إعادة ضبط الشبكة** (عربي)

يفتح نافذة طرفية تُظهر مراحل التنفيذ، ثم تُغلق تلقائياً.

### 4. اختصار لوحة المفاتيح

1. افتح **إعدادات النظام > الاختصارات** (System Settings > Shortcuts)
2. اختر **اختصارات مخصصة** (Custom Shortcuts) > **تحرير** (Edit) > **جديد** (New)
3. **الاسم:** Reset Network
4. **الأمر:** `pkexec /usr/local/bin/reset-net`
5. **اختصار لوحة المفاتيح:** `Ctrl+Alt+R` (أو أي اختصار تفضّله)

## ميزات الأمان

| الميزة | التفاصيل |
|--------|----------|
| **مسارات مطلقة** | كل أمر بمساره الكامل (`/usr/sbin/iptables` بدل `iptables`) لمنع PATH Hijacking |
| **فحوصات `-x`** | التحقق من وجود الملف وقابليته للتنفيذ قبل استخدامه |
| **`set -uo pipefail`** | منع المتغيرات غير المعرفة + فشل الأنابيب، مع عداد أخطاء بدل التوقف المفاجئ |
| **`root:root` مع `755`** | الملف مملوك للجذر فقط، لا يمكن تعديله من مستخدم عادي |
| **Polkit** | المصادقة الرسومية عبر `pkexec` بصرامة (auth_admin_keep) |
| **`timeout 10`** | جميع أوامر `systemctl restart` محمية بمهلة 10 ثوانٍ لمنع التعليق |
| **نظام لوغ** | تسجيل كل خطوة في `/var/log/reset-net.log` مع طابع زمني ومستوى الخطورة |
| **عداد أخطاء** | يكمل السكربت كل 9 خطوات ثم يُبلغ عن عدد الأخطاء في النهاية |

## سجل التنفيذ (Log)

كل عملية تسجّل تلقائياً في `/var/log/reset-net.log` مع طابع زمني ومستوى الخطورة:

```
[2026-07-01 10:30:15] [INFO] 6/9 - حذف مسارات VPN من كل الجداول والقواعد
[2026-07-01 10:30:15] [OK] تم حذف ip rule برقم 28958
[2026-07-01 10:30:15] [OK] تم تفريغ الجدول 7113
```

لمشاهدة آخر عملية:
```bash
sudo tail -20 /var/log/reset-net.log
```

أو من أيقونة العلبة: نقرة يمين > **عرض آخر سجل**.
